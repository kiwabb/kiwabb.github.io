# GC垃圾回收器
## 基本概念
### 1.什么是垃圾
**对于某个对象而言，只要应用程序中持有该对象的引用，就说明该对象不是垃圾，如果一个对象没有任何指针对其引用，它就是垃圾。**

### 2.定位垃圾的方法
1. 引用计数（ReferenceCount）

  **记录有多少个应用程序应用该对象，计数为0，说明该对象可以是垃圾。**

  `弊端`:如果出现AB对象互相引用的情况，导致永远不能回收。（循环引用导致内存泄漏）

2. 根可达算法(RootSearching)

   **通过GC Root的引用，开始向下寻找，看某个对象是否可达**
